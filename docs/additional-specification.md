# Дополнительная спецификация

[← На главную](index.html)

## Ограничения

- **Платформа:** Android 7.0 и выше (minSdk 24), целевая версия — API 36.
- **Язык и UI:** Kotlin, интерфейс полностью на Jetpack Compose.
- **Интернет:** для генерации промптов нужно подключение к сети. Без интернета
  доступен только просмотр и удаление сохранённой истории.
- **Внешний API:** приложение зависит от стороннего LLM-сервиса (OpenRouter).
  Для работы требуется действующий API-ключ; качество и скорость ответов
  зависят от сервиса.
- **Хранение данных:** все данные хранятся только локально на устройстве.
  Синхронизации между устройствами и облачного бэкапа нет.
- **Без аккаунтов:** регистрация и авторизация пользователя не предусмотрены.
- **Язык запросов:** промпты для API формируются на английском языке.

## Требования к безопасности

- Приложение запрашивает единственное разрешение — `INTERNET`. Доступ к
  контактам, файлам, геолокации и другим чувствительным данным не запрашивается.
- Персональные данные пользователя не собираются и не передаются третьим лицам.
- Все сетевые запросы выполняются по HTTPS.
- API-ключ не должен попадать в публичный репозиторий в открытом виде.
  Рекомендуется хранить ключ в `local.properties` или `BuildConfig`, а не
  жёстко в исходном коде.
- Локальная база данных доступна только самому приложению (sandbox Android).

## Требования к надёжности

- Сетевые ошибки (нет интернета, таймаут, ошибка API) перехватываются и
  показываются пользователю понятным сообщением, приложение не падает.
- Для сетевых запросов заданы таймауты (соединение — 30 с, чтение/запись — 60 с).
- Если ответ API не удаётся разобрать как корректный JSON, приложение
  обрабатывает это без аварийного завершения.
- Длительные операции выполняются в фоновых корутинах, UI-поток не блокируется,
  интерфейс остаётся отзывчивым.
- Данные сохраняются в базу сразу при нажатии **Save** и не теряются при
  закрытии приложения.

## Требования к производительности

- Запуск приложения и открытие экранов — без заметных задержек.
- Время ответа на сетевой запрос ограничено таймаутами и зависит от LLM-API
  (обычно несколько секунд); на время ожидания показывается индикатор загрузки.
- Список истории отрисовывается через `LazyColumn` — эффективно работает даже
  при большом числе сохранённых промптов.

## Требования к удобству использования

- Ответы на уточняющие вопросы выбираются в один клик.
- Удаление промпта подтверждается диалогом, чтобы избежать случайной потери.
- Интерфейс следует рекомендациям Material Design 3.
